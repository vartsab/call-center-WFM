# Capstone Report Outline

This outline is based on the engineering-project explanatory note template supplied by the program.

## Front Matter

- title page;
- thesis certification;
- academic integrity declaration;
- optional acknowledgements;
- table of contents;
- list of tables;
- list of figures;
- list of abbreviations, if needed;
- abstract.

## Abstract

Recommended length: 250-300 words.

Must include:

- project goal and relevance;
- methodology;
- main results;
- conclusions and practical value.

## Chapter 1 - Introduction

Recommended content:

- project context;
- problem statement and relevance;
- project objectives;
- significance of the project;
- structure of the explanatory note;
- chapter conclusion.

For this project, Chapter 1 should explain the call center operations problem, why interval-level forecasting and staffing matter, and what product is being developed.

## Chapter 2 - Market / Technology Environment Research

Recommended content:

- problem analysis;
- market and technology context;
- analysis of existing architectures and technologies;
- current development challenges;
- chapter conclusion.

For this project, this chapter should compare existing call center analytics, workforce management, data warehousing, forecasting, queueing theory, and scheduling approaches.

## Chapter 3 - Design Methodology

Recommended content:

- functional and non-functional requirements;
- development organization;
- technology stack justification;
- project architecture;
- implementation details for main functions;
- testing strategies;
- design challenges;
- chapter conclusion.

For this project, this chapter should describe the data pipeline, SQL Server schema, synthetic data method, forecasting method, Erlang C staffing, MILP scheduling, and Streamlit application.

## Chapter 4 - Design Results

Recommended content:

- implemented functions and project architecture;
- design/development stages;
- testing and analysis of results;
- difficulties, solutions, and future development;
- chapter conclusion.

For this project, this chapter should present screenshots, SQL outputs, forecasting metrics, staffing outputs, optimized schedule examples, and dashboard results.

## Chapter 5 - Conclusions And Recommendations

Recommended content:

- summary of completed work;
- whether objectives were achieved;
- practical impact;
- limitations;
- future improvements.

## Appendices

Possible appendices:

- SQL schema scripts;
- SQL view scripts;
- key Python code snippets;
- extended data dictionary;
- model evaluation tables;
- schedule optimization examples;
- dashboard screenshots.

## Formatting Notes

The supplied template indicates:

- final document should be PDF;
- A4 page format;
- margins: left 2.5 cm, top/bottom 2 cm, right 1.5 cm;
- Times New Roman;
- main text 12 pt;
- chapter and subsection headings bold;
- line spacing 1.5 for text, 1.0 for tables/figures/charts;
- paragraph indent 1.25 cm;
- tables, charts, and figures must be numbered within chapters and referenced in text;
- code in appendices should use Courier New 10 pt.
