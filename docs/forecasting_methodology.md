# Forecasting Methodology

## Purpose

The forecasting step predicts 30-minute inbound call volume. The forecast is then used by the Erlang C staffing calculator and the schedule optimizer.

## Dataset Expansion Decision

The project should expand forecasting history, but it should not load the full raw NYC 311 dataset into the project workspace at this stage. The more practical next step is a longer forecast-ready extract or aggregate, because raw 311 records are large and most columns are not needed for interval demand modeling.

Recommended next expansion:

- use at least 12 months of 311 demand history;
- extract only the fields needed for demand, service category, and date/time features;
- keep generated operational metadata reproducible;
- continue excluding raw and processed CSV files from version control.

## Feature Matrix

The feature matrix is generated by:

```text
src/forecasting/build_feature_matrix.py
```

It includes:

- cyclical half-hour features;
- cyclical day-of-week features;
- day, week, and month calendar features;
- weekend flag;
- US federal holiday flag;
- federal holiday name;
- distance to nearest federal holiday;
- previous-week same-interval lag.

Federal holidays are generated locally by:

```text
src/forecasting/us_federal_holidays.py
```

## Model Comparison

The current scikit-learn model comparison is generated by:

```text
src/forecasting/sklearn_model_compare.py
```

Models compared:

| Model | MAE | RMSE | MAPE |
| --- | ---: | ---: | ---: |
| Poisson regression | 1.5504 | 1.9901 | 0.5715 |
| Ridge regression | 1.6171 | 2.1294 | 0.7099 |
| Gradient boosting | 1.7169 | 2.2301 | 0.7193 |
| Histogram gradient boosting | 1.7202 | 2.2778 | 0.6974 |
| Random forest | 1.8499 | 2.4310 | 0.7644 |

The selected model is Poisson regression because it produced the lowest holdout MAE and is appropriate for count data.

## Baseline Comparison

The seasonal naive baseline achieved:

| Metric | Value |
| --- | ---: |
| MAE | 1.8299 |
| RMSE | 2.3866 |
| MAPE | 0.7344 |

The feature-based Poisson model improves all three metrics on the current sample. This should be treated as an initial result because the current training history is only January 2025.
